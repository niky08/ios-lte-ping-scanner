# LTE Ping Scanner (iOS, 2026)

Простое iOS-приложение для проверки доступности IP-диапазонов по ICMP (LTE / L3 whitelist).

## Сборка

### Локально (Xcode)

1. Откройте `PingScanner.xcodeproj` в Xcode 16+
2. Выберите свой **Team** для подписи
3. Соберите на **реальном iPhone** (ICMP не работает в Simulator)
4. Запускайте скан на **LTE**, не Wi‑Fi

### CI (GitHub Actions)

1. **Actions** → **Build iOS IPA (LTE Ping Scanner)** → **Run workflow**
2. Скачайте артефакт `lte-ping-scanner-unsigned-ipa`
3. Подпишите IPA своим сертификатом (AltStore, Sideloadly и т.п.)

## Использование

- Введите маску: `111.88.x.x`, `111.88.1.x` или диапазон `111.88.1.1-111.88.1.50`
- Нажмите **Сканировать**
- **Ответил** = диапазон/хост доступен на L3
- **Нет ответа** = вероятно заблокировано оператором (или хост молчит)

## Рекомендуемые настройки ping (2026)

| Параметр | По умолчанию | Комментарий |
|----------|--------------|-------------|
| TTL | 64 | Стандарт |
| Timeout | **150 ms** | При RTT 30–50 ms: 100 ms даёт ложные «мёртвые»; 150–200 ms надёжнее |
| Packet size | 56 B | Достаточно для ICMP echo |
| Параллельно | 48 | Быстрее чем interval 0.1 последовательно |

Для первого прогона лучше **`111.88.1.x`** (~254 IP), не весь `/16` (~65k).

## Ограничения iOS

- ICMP требует реального устройства
- Оператор может фильтровать ICMP — тогда «нет ответа» не всегда значит блок L3
- Большие диапазоны долго грузят CPU/радио
